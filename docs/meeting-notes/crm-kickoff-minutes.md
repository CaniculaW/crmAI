## V1 范围

V1 目标是先让销售基础过程可上线试点，覆盖以下模块：

1. 系统基础与权限管理：登录、登出、会话、密码、组织、用户、角色、权限、数据权限、字典配置、审计日志。
2. 客户池：客户新建/编辑、列表筛选、详情查看、集团/子公司关系、负责人和协同人员、最近跟进回写。
3. 联系人与干系人：联系人档案、项目角色、影响力、态度、关系热度、关联商机、客户关系地图或关系分组视图。
4. 商机生命周期：商机新建/编辑、阶段推进、状态管理、关闭/取消跟进、风险管理、最近跟进回写。
5. 销售行动：行动新建/编辑、行动状态、下一步计划、风险记录、客户/商机回写、我的待办。
6. 周进展：由已完成销售行动自动汇总，支持按商机、负责人、自然周查看。

## V1 不含范围

以下能力不进入 V1 开发，统一进入后续版本池：

- 方案与标书模块。
- 合同管理模块。
- 开票管理模块。
- 回款管理模块。
- 发票与回款多对多核销。
- 经营驾驶舱完整指标、合同/开票/回款看板、完整风险预警。
- AI 销售助手完整能力，包括文本录入、草稿生成、缺失字段追问、AI 写入日志等。
- ERP、电子签章、税控开票、复杂 CPQ、原生移动端完整实现。

## 会议结论

1. V1 以销售基础闭环为唯一上线目标，范围待启动会最终确认后冻结。
2. V2/V3/V4 能力不进入 V1 开发，统一进入后续版本池。
3. 技术栈按启动默认决策进入工程骨架：React + Ant Design、Java Spring Boot、PostgreSQL；研发负责人可在 Sprint 0 评审中发起变更。
4. 销售侧验收人和管理侧验收人必须在 Sprint 0 完成指定。
5. 页面清单、低保真原型、DDL/API/测试清单完成后进入 Sprint 1 开发排期。
