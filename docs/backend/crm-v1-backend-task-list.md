# CRM V1 后端开发任务清单

日期：2026-06-17

适用范围：按研发启动计划Task 6拆分后端Sprint任务。默认技术栈参考 `docs/architecture/tech-stack-decision.md`：Java Spring Boot + PostgreSQL。

## 1. Sprint执行原则

- V1后端优先跑通闭环：登录 -> 创建客户 -> 创建联系人 -> 创建商机 -> 创建销售行动 -> 完成行动 -> 回写最近跟进 -> 查看周进展。
- 每个任务必须包含接口、服务逻辑、数据权限、参数校验、审计或日志、单元/集成测试。
- 所有业务列表必须默认过滤软删除数据。
- 关键操作不得只依赖前端权限控制。

## 3. 模块验收口径

### 3.1 工程骨架

- 本地服务可以启动。
- 支持数据库连接。
- 提供健康检查接口。
- 支持基础日志、trace_id、统一异常处理。

### 3.2 认证与权限

- 用户可以登录和登出。
- 停用用户不能登录。
- 用户可以修改密码。
- 管理员可以重置密码。
- 接口可以校验登录态。
- 接口可以校验角色、权限点和数据范围。
- 关键操作写入审计日志。

### 3.3 客户池

- 客户可新增、编辑、查询、详情查看。
- 支持上级客户。
- 支持负责人、归属部门、协同人员。
- 支持最近跟进回写。

### 3.4 联系人与干系人

- 联系人可新增、编辑、查询、详情查看。
- 支持关联客户和商机。
- 支持项目角色、影响力、态度、关系热度。
- 支持商机决策链查询。

### 3.5 商机

- 商机可新增、编辑、查询、详情查看。
- 阶段和状态独立维护。
- 关闭和取消跟进必须填写原因。
- 关闭和取消跟进商机不进入默认跟进列表。

### 3.6 销售行动

- 行动可新增、编辑、查询、完成。
- 行动可关联客户、商机、联系人。
- 行动完成后回写客户和商机最近跟进。
- 风险行动可更新商机风险状态。

### 3.7 周进展汇总

- 商机周进展来自已完成销售行动。
- 支持按自然周查询。
- 同一商机一周多条行动保留多条明细。
- 商机主表不存储第一周、第二周、第三周、第四周进展字段。

## 4. Sprint建议拆分

| Sprint | 建议任务 |
|---|---|
| Sprint 0 | BE-001至BE-006，完成工程、迁移、字典、权限基础模型。 |
| Sprint 1 | BE-007至BE-013，完成认证权限、客户、联系人、商机基础闭环。 |
| Sprint 2 | BE-014至BE-018，完成行动、回写、周进展、附件和提醒。 |
| Sprint 3 | BE-019至BE-020，联调、回归、性能和试点修复。 |

## 5. 待确认项

- 会话机制采用JWT还是服务端Session。
- 持久层采用JPA还是MyBatis Plus。
- 附件上传是否在V1实现完整文件链路，还是只做元数据。
- 是否需要导出接口进入V1。
- 数据权限中“协同数据”是否覆盖客户协同、商机协同、行动参与人三类。
