# V1 商机推进链路页面逻辑设计

日期：2026-06-25

## 目标

把商机页从“商机数据表”调整为“商机推进工作台”。销售进入页面后，优先理解这条商机处于什么阶段、是否还在推进、有什么风险、最近做了什么、下一步应该做什么。

## 页面范围

- 商机列表。
- 商机详情抽屉。
- 商机新建与编辑。
- 商机阶段、状态、风险、关闭/取消和重启入口。
- 关联客户、联系人、销售行动、周进展入口。

## 不做范围

- 不新增报价、合同、收款、开票。
- 不调整后端模型。
- 不把商机扩展成项目交付管理。
- 不引入 V2 的商机预测或经营分析能力。

## 方案 B：商机推进工作台

采用列表 + 推进入口抽屉的结构：

- 列表聚焦在办商机：商机名称、所属客户、阶段、状态、风险、最近跟进。
- 详情抽屉展示推进判断：所属客户、阶段、状态、风险、等级、预计合同金额。
- 详情抽屉展示推进记录：当前进展、下一步计划、最近行动。
- 详情抽屉给出关联入口：查看客户、经营联系人、记录销售行动、查看周进展。
- 关闭/取消与重启继续保留在列表操作中。

## 验收点

- 用户从商机列表点商机名称或“查看推进”，能进入“商机推进入口”。
- 详情中不出现原始枚举作为主要判断信息，阶段、状态、风险需要中文化。
- 详情中能看到当前进展、下一步计划、最近行动。
- 详情中能跳转客户、联系人、销售行动、周进展。
- 本地 UAT 页面无服务端异常横幅，无控制台应用错误。
