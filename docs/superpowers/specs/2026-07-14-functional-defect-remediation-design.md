# CRM AI 功能缺陷修复设计

日期：2026-07-14  
范围：2026-07-14 公网功能测试发现的 3 个 P1、7 个 P2，以及测试工具清理问题。

## 1. 目标

将当前本地源码作为唯一修复基线，补齐仍存在的业务校验、错误映射、权限路由和 AI 降级说明，然后以同一提交重新构建前后端并升级公网数据库到最新 Flyway 版本。修复完成后，公网必须通过缺陷定点回归和核心业务链路回归。

## 2. 基线判断

- 公网前端、后端和数据库均停留在 Flyway V35 对应版本。
- 当前本地源码已包含 V36-V38 审批流、移动端抽屉导航和会话读取时的用户状态校验。
- 因此“审批不可用”“移动端只有菜单”“停用用户旧会话有效”先按版本升级项处理，不重复实现；升级前后分别用现有复现用例确认。
- 当前源码仍缺少合同金额正数校验、统一 4xx 异常映射、行动重复完成限制、仪表盘日期范围校验、系统管理路由守卫和 AI 规则降级说明。

## 3. 方案比较

### 方案 A：直接修补公网容器

优点是短期速度快。缺点是无法形成可追溯源码，下一次构建会覆盖修补，数据库和制品仍可能漂移。不采用。

### 方案 B：源码修复、完整回归、同提交部署

先为每个缺陷补失败测试，再修改源码；构建同一提交的前后端镜像，执行 Flyway V36-V38 及新增迁移，最后在公网复验。可追溯、可回滚，采用此方案。

### 方案 C：同步重构异常、权限和 AI 架构

长期结构更整齐，但会扩大本轮改动面并延迟恢复可用性。不在本轮实施，只做必要的局部抽象。

## 4. 修复设计

### 4.1 数据完整性

- 合同创建和更新请求的 `contract_amount` 必须大于 0。
- 服务层保留同样的正数校验，避免绕过 Bean Validation。
- 新增数据库检查约束迁移，阻止非应用入口写入无效金额。
- 部署前只清理本轮明确标记的负金额测试合同，不修改真实业务数据。

### 4.2 API 错误契约

- 未匹配的 `/api/**` 路由返回 404 和 `NOT_FOUND`。
- 资源不存在使用专用 `ResourceNotFoundException`，返回 404。
- 参数格式错误和普通 `IllegalArgumentException` 返回 400。
- 业务状态冲突继续使用 `BusinessRuleException`，返回 409。
- 所有错误保留 Trace ID，不向客户端暴露堆栈。

### 4.3 状态机和日期范围

- 销售行动仅允许未完成状态进入完成操作；重复完成返回 409，不更新完成时间、不重复写审计和业务回填。
- 仪表盘所有含 `date_from`、`date_to` 的入口共用日期范围校验；开始日期晚于结束日期返回 400。

### 4.4 权限路由

- 系统管理二级路由在渲染页面之前校验对应权限。
- 无权限直接访问时跳转到系统概览或工作台，不渲染标题、表单和操作按钮。
- 后端 403 继续作为最终安全边界，前端守卫负责避免误导和信息暴露。

### 4.5 AI 降级透明度

- AI 上下文摘要返回明确的生成模式：`openai` 或 `rules_fallback`。
- 无可用模型配置时返回稳定的中文说明，明确当前使用规则辅助模式、不会调用远程模型。
- AI 工作台及生成页面展示同一状态提示；不阻断规则模式现有能力。
- 日志和生成结果保持现有人工确认要求。

### 4.6 部署一致性

- 前后端由同一 Git 提交构建。
- 后端启动后 Flyway 必须达到最新版本，审批权限必须进入管理员权限集。
- 部署验证必须同时检查前端制品标识、健康接口、Flyway 版本和审批路由，任一不一致即回滚。

### 4.7 测试工具

- Chrome smoke 清理临时目录时增加重试，避免应用断言通过却因目录占用返回失败。
- 测试工具问题单独记录，不与产品缺陷混淆。

## 5. 测试策略

- 每个源代码缺陷先新增失败测试，再实施修复。
- 后端执行完整 `mvn test`，前端执行完整 `npm test` 和 `npm run build`。
- 本地 PostgreSQL 执行 Flyway 集成验证。
- 公网升级后重跑 10 条缺陷复现用例、V4 API smoke、系统管理浏览器 smoke、审批全链路及 RBAC 新账号登录。
- 准出条件保持：P0/P1 为 0，核心链路 100% 通过，全部功能用例通过率不低于 95%。

## 6. 非目标

- 本轮不重构大型 `App.tsx`。
- 不新增审批撤回能力；当前产品契约没有该接口，后续单独立项。
- 不接入本地模型，不扩大 OpenAI 模型配置范围。
- 不执行性能和安全专项测试；功能回归通过后再进入后续测试阶段。
