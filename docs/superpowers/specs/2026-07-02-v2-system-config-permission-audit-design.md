# V2 系统配置与权限审计设计说明

日期：2026-07-02

## 1. 目标

模块 8 不重做系统管理页面，而是在现有“系统”一级菜单下补齐 V2 治理能力：字典、权限点、角色授权、审计日志和数据权限验证。目标是让销售、商务、财务、管理员在 V2 销售到财务闭环中看到合适入口，关键金额/状态/核销动作可追溯，最终全链路 UAT 前没有治理缺口。

## 2. 现状判断

当前系统管理已经拆成独立页面：

- `系统概览`
- `组织管理`
- `用户管理`
- `角色权限`
- `审计日志`
- `字典管理`

后端已经具备基础能力：

- `IdentityAdminController` 管理组织、用户、角色、权限。
- `AuditLogController` 支持按模块、动作、对象、操作人和数量筛选审计日志。
- `DictionaryController` 支持字典类型、字典项新增和字典项启停。
- V2 业务模块已有方案、合同、开票、回款、核销、附件相关权限点。
- V2 业务动作大多已写审计，包括方案作废、合同终止、开票申请/登记/异常/作废、回款确认/异常/退款、核销/撤销等。

明确缺口：

- 字典管理写操作目前有权限校验，但没有审计记录。
- 角色授权弹窗以平铺权限为主，V2 权限点不够容易按模块检查。
- 审计日志前端只展示最近记录，缺少 V2 快速筛选和字段筛选入口。
- 数据权限已有服务，但需要在 V2 全链路补一组回归测试证明各模块遵守客户/合同关联的数据可见性。

## 3. 推荐方案

采用方案 B：系统作为一级菜单，下面保留独立专业页面，并补齐 V2 治理覆盖。

不把组织、用户、角色权限、审计日志、字典揉回一个大页面。专业系统中这些能力通常是独立页面，便于授权隔离、审计追溯和管理员日常维护。V2 本轮只做治理补齐，不引入复杂 IAM 产品形态。

## 4. 页面与交互边界

### 4.1 系统概览

系统概览继续作为入口页，展示组织、用户、角色权限、审计日志、字典管理五个卡片。模块 8 增强后，概览需要能提示 V2 治理覆盖状态：

- V2 权限点覆盖数。
- V2 字典类型覆盖数。
- 最近 V2 审计记录数。

### 4.2 角色权限

角色权限页保持“角色列表 + 授权弹窗”模式。授权弹窗增强为按模块分组展示：

- V1 基础业务。
- V2 方案标书。
- V2 合同。
- V2 开票。
- V2 回款。
- V2 核销。
- 附件。
- 系统管理。

每个权限仍然使用既有 `permission_code`，不新增前端自定义授权模型。

### 4.3 审计日志

审计日志页保留表格，新增筛选区：

- 模块编码：`solution`、`contract`、`invoice`、`receivable`、`payment`、`reconciliation`、`attachment`、`system`。
- 动作编码：自由输入或选择常见 V2 动作。
- 对象类型。
- 对象 ID。
- 操作人 ID。
- 返回条数。

同时提供 V2 快速筛选：

- 方案审计。
- 合同审计。
- 开票审计。
- 回款审计。
- 核销审计。
- 字典/角色变更。

### 4.4 字典管理

字典管理页保持字典卡片模式，补齐 V2 字典覆盖。V2 至少覆盖：

- 方案/标书：材料类型、方案状态、自评结果、风险等级。
- 合同：合同类型、合同状态、变更类型、节点状态。
- 开票：开票状态、发票类型、发票异常类型。
- 回款：回款计划状态、到账状态、回款方式、跟进结果。
- 核销：核销状态、核销来源。

字典写操作必须进入审计日志：

- `system.dict.type.create`
- `system.dict.item.create`
- `system.dict.item.update`

### 4.5 数据权限

V2 不做行级规则可视化配置器。继续复用 `sys_data_scopes`、`sys_role_data_scopes` 和 `DataPermissionService`，通过用户角色的数据范围控制客户、商机、方案、合同、开票、回款、核销可见性。

模块 8 需要补回归测试，证明：

- 方案按客户/商机归属过滤。
- 合同按客户/商机/合同归属过滤。
- 开票、回款、核销通过合同归属间接受控。
- 拥有 `global` 范围的角色可以看全量。
- 无数据范围的角色看不到受控业务数据。

## 5. 后端边界

新增或调整：

- 新增迁移 `V20__complete_v2_system_governance.sql`，补齐 V2 字典种子数据和必要权限/角色授权兜底。
- `DictionaryController` 注入 `AuditLogService`，在字典类型新增、字典项新增、字典项更新后写审计。
- `DictionaryControllerTest` 增加字典写操作审计断言。
- `DatabaseMigrationTest` 增加 V2 字典覆盖断言。
- V2 相关 Controller 测试补数据权限回归，不新增新的权限框架。

不做：

- 不做 SSO、LDAP、OAuth。
- 不做字段级权限。
- 不做行级规则可视化配置。
- 不做审计归档、导出、外部 SIEM 对接。

## 6. 前端边界

调整 `frontend/src/App.tsx` 中 `SystemPage`，保持现有页面结构：

- 角色授权弹窗按权限模块分组。
- 审计日志页新增筛选表单和 V2 快速筛选。
- 字典管理文案从“V1 基础选项”调整为“基础选项与 V2 商务财务选项”。
- 系统概览增加 V2 治理覆盖摘要。

新增或调整 `frontend/src/App.test.tsx`：

- 验证角色授权按 V2 模块分组展示。
- 验证审计日志筛选会调用 `/api/system/audit-logs` 并带上筛选参数。
- 验证系统概览展示 V2 治理覆盖摘要。

## 7. 验收标准

- V2 字典类型和关键字典项在迁移后存在。
- V2 权限点按模块可见，角色授权仍可保存。
- 字典新增/更新、角色授权、组织/用户变更可在审计日志中追溯。
- 审计日志前端可按 V2 模块快速筛选。
- V2 业务数据权限回归测试通过。
- 后端定向测试、前端测试和构建通过。
- 浏览器 UAT 能展示系统概览、角色权限、审计日志、字典管理，并无服务端异常、无控制台应用错误。
