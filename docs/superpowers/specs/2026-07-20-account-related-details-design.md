# 客户详情关联联系人与商机设计

## 目标

优化“客户池 > 客户经营”详情抽屉，使销售人员无需离开客户上下文即可查看该客户关联的联系人和商机，并能通过稳定 URL 跳转到对应联系人详情和商机推进详情。

## 范围

本次包含：

- 客户经营抽屉新增“关联联系人”和“关联商机”两个区域。
- 联系人和商机名称支持点击跳转到各自详情。
- 联系人页面新增 `contact_id` 查询参数深链能力。
- 商机页面复用已有 `opportunity_id` 查询参数深链能力。
- 按当前用户权限决定是否请求和展示关联数据。
- 增加自动化测试和桌面、移动端浏览器验收。

本次不包含：

- 新建独立客户详情路由。
- 修改后端数据结构或新增聚合接口。
- 在客户详情内编辑联系人或商机。
- 将商机重命名为项目。

## 页面结构

客户经营抽屉保留现有客户摘要、最近跟进、业务入口、V2 业务闭环和下一步建议。在“最近跟进”之后增加两个关联区域：

### 关联联系人

- 标题右侧显示记录数和“查看全部”。
- 表格字段：姓名、职务、联系人类型、态度、关系热度。
- 姓名链接格式：`/contacts?account_id={accountId}&contact_id={contactId}`。
- “查看全部”链接格式：`/contacts?account_id={accountId}`。
- 联系人页面收到 `contact_id` 后请求联系人详情并自动打开“联系人经营”抽屉。

### 关联商机

- 标题右侧显示记录数和“查看全部”。
- 表格字段：商机名称、阶段、状态、风险、预计合同金额。
- 商机名称链接格式：`/opportunities?account_id={accountId}&opportunity_id={opportunityId}`。
- “查看全部”链接格式：`/opportunities?account_id={accountId}`。
- 商机页面继续使用现有 `opportunity_id` 详情加载和自动打开能力。

两个表格使用紧凑尺寸、不分页，并限制在客户详情抽屉内。移动端宽度不足时仅表格容器横向滚动，不产生页面级横向溢出。

## 组件边界

新增独立的 `AccountRelatedRecords` 组件，由 `AccountOperationDrawer` 在客户存在时渲染。该组件只负责：

- 根据客户 ID 和权限加载联系人、商机。
- 渲染两个关联表格及状态。
- 生成列表和详情深链。

这样可避免在当前会对空客户直接返回的 `AccountOperationDrawer` 中引入条件 Hook，并将关联数据逻辑与客户摘要逻辑隔离。

联系人页面增加查询参数初始化和详情加载逻辑：

- `useInitialQueryFilters(["account_id", "contact_id"])` 解析地址。
- 列表请求只保留 `account_id`，不向后端传递 `contact_id`。
- 有有效 `contact_id` 时调用 `crmApi.contacts.detail(id)`，成功后设置为当前联系人。

## 数据与权限

沿用现有接口：

- `GET /api/contacts?account_id={accountId}`
- `GET /api/opportunities?account_id={accountId}`
- `GET /api/contacts/{contactId}`
- `GET /api/opportunities/{opportunityId}`

权限规则：

- 仅有 `contact.read` 时加载并展示关联联系人。
- 仅有 `opportunity.read` 时加载并展示关联商机。
- 没有对应权限时不发起接口请求，并隐藏对应区域。
- 两项权限都没有时不渲染关联数据容器。

## 状态与错误处理

- 加载中：对应区域显示表格加载状态。
- 空数据：分别显示“暂无关联联系人”和“暂无关联商机”。
- 单项失败：只在失败区域显示错误提示，另一项数据仍可正常使用。
- 深链详情失败：目标列表页保留，展示现有页面错误状态，不打开错误详情抽屉。
- 客户切换：关联组件以客户 ID 为依赖重新请求，旧数据不作为新客户结果展示。

## 测试策略

### 组件与页面测试

- 打开客户经营后，验证按客户 ID 请求联系人和商机。
- 验证联系人、商机字段和记录数展示。
- 验证联系人详情、商机详情及“查看全部”链接包含正确查询参数。
- 验证缺少权限时不请求、不展示对应区域。
- 验证联系人 `contact_id` 深链会请求详情并自动打开联系人经营抽屉。

### 回归与浏览器验收

- 运行完整前端 Vitest 测试。
- 运行 TypeScript 和 Vite 生产构建。
- 在 1440x1000 和 390x844 下验证客户详情、联系人详情跳转和商机详情跳转。
- 检查控制台错误、失败接口、抽屉内表格溢出和移动端页面级横向溢出。

## 验收标准

- 客户详情正确展示当前客户关联的全部可读联系人和商机。
- 点击联系人姓名进入联系人页面并自动打开对应联系人详情。
- 点击商机名称进入商机页面并自动打开对应商机详情。
- “查看全部”进入带客户筛选条件的列表页。
- 权限隔离、加载、空数据和单项错误状态符合设计。
- 现有客户、联系人、商机创建、编辑和查询行为无回归。
