# V1 周进展链路页面逻辑设计

日期：2026-06-26

## 目标

把周进展页从“周进展数据表”调整为“周进展复盘入口”。销售或管理者进入页面后，能按自然周理解某个客户和商机本周发生了什么、完成了哪些行动、下一步是什么、是否存在风险，并能追溯回客户、商机和销售行动。

## 页面范围

- 周进展列表。
- 周进展复盘抽屉。
- 自然周、负责人、客户、商机、风险筛选。
- 周进展行动明细。
- 关联客户、商机、销售行动入口。

## 不做范围

- 不做 BI 报表、绩效考核、预测分析。
- 不新增月报、季报、管理驾驶舱。
- 不调整后端模型。
- 不新增 V2 合同、报价、开票、回款能力。

## 方案 B：周进展复盘入口

采用列表 + 复盘抽屉的结构：

- 列表展示客户、商机、负责人、自然周、行动数、最近行动、风险状态。
- 商机名称可点击进入“周进展复盘入口”。
- 复盘抽屉展示复盘摘要：客户、商机、负责人、本周周期、行动数量、最近行动时间、风险状态。
- 复盘抽屉展示行动明细卡片：行动主题、结论、下一步、风险说明、行动时间、行动结果。
- 复盘抽屉给出关联入口：查看客户、推进商机、查看销售行动。

## 验收点

- 用户从周进展列表点商机名称，能进入“周进展复盘入口”。
- 列表不再主要显示客户 ID、商机 ID，而是显示客户名称和商机名称。
- 详情中能看到自然周、客户、商机、负责人、行动数、最近行动时间和风险状态。
- 行动明细能清楚展示主题、结论、下一步和风险说明。
- 详情中能跳转客户、商机和销售行动。
- 本地 UAT 页面无服务端异常横幅，无控制台应用错误。
