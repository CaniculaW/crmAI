# CRM AI 全局紧凑型界面设计

日期：2026-07-20

## 1. 目标

参考用户提供的专业业务系统界面，将 CRM AI 从当前偏宽松的页面密度调整为紧凑、清晰、适合高频业务操作的工作台风格。调整覆盖驾驶舱、业务列表、系统管理、审批、AI 页面、表单、弹窗和抽屉；登录页保持现状。

本次只调整视觉密度和字体层级，不改变业务结构、菜单层级、路由、权限、字段和交互逻辑。

## 2. 设计原则

- 信息优先：首屏展示更多表格行、筛选项和业务信息。
- 层级清晰：页面标题、区块标题、正文和辅助文字保持稳定字号梯度。
- 全局一致：通过全局样式和 Ant Design 组件覆盖统一生效，不逐页堆叠例外。
- 可操作性：桌面端紧凑，移动端仍保留足够的触控高度和换行空间。
- 低风险：沿用现有颜色、图标、圆角和组件结构，避免无关重构。

## 3. 全局密度规范

| 元素 | 当前典型值 | 目标值 |
|---|---:|---:|
| 正文 | 14px | 13px |
| 页面标题 | 24px | 20px |
| 区块标题 | 18px | 16px |
| 辅助文字 | 12-13px | 12px |
| 顶栏高度 | 64px | 52px |
| 桌面侧栏宽度 | 232px | 216px |
| 页面内边距 | 24px | 18px |
| 页面区块间距 | 14-20px | 10-12px |
| 卡片内边距 | 18-24px | 14px |
| 常规控件高度 | 32-40px | 32px |
| 表格单元格纵向内边距 | Ant 默认 | 9-10px |
| 菜单项高度 | Ant 默认 | 40px |

内容区保持现有最大宽度约束，但在宽屏允许列表与工作台充分使用可用空间。圆角维持不超过 8px，不改变现有蓝色主色和中性色体系。

## 4. 组件调整

### 4.1 应用框架

- 侧栏收窄到 216px，品牌区和菜单缩短纵向留白。
- 顶栏固定为紧凑单行布局，桌面端高度约 52px。
- 内容区边距改为 18px，移动端使用 12px。
- 保持侧栏固定、右侧内容独立滚动的现有行为。

### 4.2 标题与说明

- 页面标题使用 20px、较粗字重。
- 页面说明使用 12px，并压缩标题与说明之间距离。
- 页面标题区与正文间距降至 12px。
- 引导条、错误提示和健康提示统一使用紧凑内边距。

### 4.3 表格与筛选

- 表格正文和表头统一为 13px，表头保持中等字重。
- 表头与数据行纵向内边距控制在 9-10px。
- 筛选栏控件高度统一为 32px，字段间距缩短。
- 操作列按钮使用小尺寸，长字段允许省略或自然换行，不压缩到不可读。

### 4.4 卡片、驾驶舱与 AI 页面

- 卡片 body 内边距统一约 14px，标题字号使用 15-16px。
- 指标卡高度和数值字号适度收缩，但保留数字视觉优先级。
- 驾驶舱网格间距降至 10-12px。
- AI 工作台的快捷任务、证据、建议和确认区采用同一密度，不改变人工确认流程。

### 4.5 表单、弹窗与抽屉

- 输入框、下拉框、日期选择器和按钮以 32px 为常规高度。
- 表单项底部间距缩至 12px，标签使用 13px。
- 弹窗与抽屉 body 内边距缩至 16-20px。
- 详情区块间距和记录字段间距同步收缩，仍保证错误文本不会与下一字段重叠。

## 5. 响应式边界

- 桌面端（宽度大于 900px）：使用完整紧凑规范和 216px 固定侧栏。
- 平板与移动端（宽度不大于 900px）：侧栏改为抽屉，内容边距 12px。
- 移动端按钮、输入框和菜单项最小交互高度为 36px。
- 筛选器、标题操作区和表格工具栏允许换行，不使用视口宽度缩放字体。
- 表格可横向滚动，但页面主体不得产生非预期横向溢出。

## 6. 实施范围

主要修改 `frontend/src/styles.css`，必要时在应用入口补充 Ant Design 紧凑主题变量。仅当全局样式无法稳定覆盖时，才对共享的 `PageTitle`、`DataWorkspace` 或应用框架增加少量语义类名。

不调整：

- 登录页视觉结构。
- 页面信息架构和业务字段。
- 路由与权限判断。
- 数据接口与后端代码。
- 当前主色、状态色和图标体系。

## 7. 验收标准

- 客户、合同、审批中心、系统用户、经营驾驶舱五类代表页面符合紧凑型业务系统观感。
- 桌面端 1440x1000 可视区域展示的信息量明显增加。
- 移动端 390x844 无标题、按钮、表单、弹窗和导航重叠。
- 所有文字无裁切；长名称可省略或换行，不能撑破容器。
- 固定侧栏与右侧独立滚动行为保持正常。
- 前端单元测试、生产构建和浏览器多视口 smoke 全部通过。

## 8. 风险控制

- 全局 Ant Design 覆盖可能影响少数复杂组件，因此先建立代表页面视觉基线，再逐项调整覆盖选择器。
- 紧凑表格可能造成操作列拥挤，操作按钮保留最小点击区域，并检查长中文字段。
- 移动端不直接继承桌面控件高度，通过媒体查询恢复到至少 36px。
