# V1 Contact Relationship Chain Design

日期：2026-06-25

## 目标

把 V1 联系人页从“联系人数据表”重构为“客户关系经营入口”，帮助最终确认人理解：这个联系人属于哪个客户、在项目中扮演什么角色、当前态度和关系热度如何、下一步应该怎么经营。

## 已确认方案

采用方案 B：联系人关系经营入口。

- 保留现有列表、筛选、新建、编辑和后端 API。
- 中文化项目角色、态度、关系热度、联系人类型等枚举。
- 将关系视图升级为“关键关系判断”，覆盖项目角色、态度、关系热度三组。
- 将联系人详情改为“联系人经营入口”，展示所属客户、角色定位、态度热度、最近沟通、下一步动作和业务入口。
- 业务入口指向客户池、商机、销售行动，保持 V1 范围内的链路闭环。

## 不做范围

- 不调整后端模型。
- 不新增 V2 合同、回款、开票能力。
- 不新增社交图谱或复杂关系网络算法。
- 不做完整客户组织树。

## 页面行为

联系人列表继续用于检索和操作，但字段展示要面向业务判断。联系人详情抽屉不再只是字段清单，而是按照“关系对象 -> 关系判断 -> 最近沟通 -> 后续动作”组织。

关系视图保留分组能力，但分组名称要中文化，并补充“谁是决策人、谁支持、谁需要经营”的判断语义。没有数据时保留现有空状态。

## 验收标准

- 联系人详情能看到“联系人经营入口”。
- 详情中能看到所属客户、项目角色、态度、关系热度、最近沟通和下一步动作。
- 详情中有“查看客户”“推进商机”“记录销售行动”入口。
- 列表与关系视图不再裸露 `supporter`、`trusted`、`decision_maker` 等英文枚举。
- `npm test` 和 `npm run build` 通过。
- 本地 UAT `/contacts` 无服务端异常横幅，无控制台应用错误。
