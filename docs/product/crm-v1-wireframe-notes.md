# CRM V1 低保真原型说明

版本：V1.0

日期：2026-06-17

状态：Sprint 0 产品评审稿。本文用文字区块描述低保真布局，不绘制复杂图形；视觉稿与交互细节待 UI/UX 继续细化。

## 1. 全局布局原则

- Web 管理端优先，采用左侧导航 + 顶部用户区 + 主内容区布局。
- V1 菜单包括：工作台、客户池、联系人与干系人、商机、销售行动、周进展、系统管理。
- 列表页统一结构：顶部标题与主操作按钮，其下为筛选栏，再下为表格、分页和批量操作占位。
- 详情页统一结构：顶部摘要区 + 主操作按钮 + Tab 内容区。
- 表单页统一结构：基础信息、业务字段、关系字段、附件/备注四类分组；保存、取消固定在底部或右下角。
- 权限控制在菜单、按钮和数据三层体现；不可操作内容不显示或置灰，具体策略待研发确认。

## 2. 登录与密码

### 2.1 登录页

```text
[页面容器]
  [左侧品牌/系统名称区域]
    - 项目型大客户CRM
    - 销售基础闭环试点系统

  [右侧登录表单]
    - 账号输入框
    - 密码输入框
    - 登录按钮
    - 忘记密码入口
    - 登录错误提示区域
```

交互说明：

- 登录成功后进入工作台。
- 停用用户、密码错误、会话异常需要展示明确错误提示。
- 验证码、单点登录、二次认证均为待确认，不作为 V1 默认必做。

### 2.2 忘记密码页

```text
[标题：找回或重置密码]
  [说明区]
    - 根据企业安全策略，支持自助重置或联系管理员重置，具体方式待确认

  [表单区]
    - 账号
    - 邮箱/手机号，待确认
    - 提交按钮
    - 返回登录
```

交互说明：

- 不通过页面暴露账号是否存在等敏感信息。
- 若 V1 只支持管理员重置，此页作为引导页保留。

### 2.3 修改密码页

```text
[标题：修改密码]
  [表单区]
    - 旧密码
    - 新密码
    - 确认新密码
    - 密码强度提示
    - 保存按钮
```

交互说明：

- 新密码和确认新密码需一致。
- 密码强度规则使用默认建议，具体长度与复杂度待技术栈决策文档确认。

```text
    - 本周行动数
    - 高风险商机数

    [我的待办]

      - 逾期行动
      - 下次跟进提醒

    [我的商机]
      - 商机名称
      - 客户
      - 阶段/状态

  [右侧栏]
```

- 待办、今日行动、本周行动均可下钻到行动详情。

## 4. 客户详情

### 4.1 客户详情布局

```text
[顶部摘要区]
  - 客户名称 / 客户等级 / 客户状态
  - 所属行业 / 区域 / 负责人 / 协同人员
  - 最近跟进时间 / 最近跟进摘要
  - 主操作：编辑客户、新建联系人、新建商机、新建销售行动

[Tab 区]
  [基础信息]
    - 客户类型、上级客户、客户来源、归属部门
    - 客户背景、重点需求方向、关系状态、下次跟进时间

  [联系人/干系人]
    - 联系人表格
    - 关键字段：姓名、部门、职务、类型、影响力、态度、关系热度
    - 操作：新建联系人、查看详情

  [商机]
    - 商机表格
    - 关键字段：商机名称、阶段、状态、预计金额、负责人、风险、最近跟进
    - 操作：新建商机、查看详情

  [销售行动]
    - 行动时间线或表格
    - 关键字段：行动主题、类型、状态、结果、下一步计划、风险
    - 操作：新建行动、查看详情

  [附件]
    - 客户资料、组织架构、历史材料，V1 附件能力可作为基础入口预留
```

验收关注：

- 客户详情必须体现客户 -> 联系人 -> 商机 -> 行动的上下游关系。
- 销售行动完成后，客户最近跟进时间和摘要自动更新。
- 方案、合同、开票、回款 Tab 不进入 V1；如需展示，仅作为后续版本入口占位，默认不开发。

## 6. 商机详情

### 6.1 商机详情布局

```text
[顶部摘要区]
  - 商机名称 / 所属客户
  - 商机阶段 / 商机状态 / 风险状态
  - 预计预算 / 预计合同金额 / 赢率 / 预计成交时间
  - 责任人 / 协同人员 / 最近跟进
  - 主操作：编辑商机、推进阶段、变更状态、新建销售行动、关闭/取消跟进

[阶段与状态区]
  - 阶段条：商业线索 -> 商业验证 -> 商业方案 -> 商业谈判 -> 商业成交
  - 状态标签：跟进中 / 暂停 / 关闭 / 取消跟进
  - 阶段必填项缺失提示

[Tab 区]
  [基础信息]
    - 来源、机会点、需求、预算、采购流程、资金来源、项目周期

  [阶段与状态]
    - 阶段推进记录
    - 状态变更记录
    - 关闭/取消跟进原因

  [干系人]
    - 关联联系人
    - 项目角色、影响力、态度、关系热度
    - 关系缺口提示

  [销售行动]
    - 行动列表
    - 支持新建行动
    - 支持按状态、类型、风险筛选

  [周进展]
    - 本周/历史自然周
    - 行动时间、行动主题、形成结论、下一步计划、风险说明

  [风险]
    - 风险状态
    - 风险说明
    - 来源销售行动

  [附件]
    - 需求资料、纪要等基础附件入口
```

### 6.2 商机关闭/取消跟进弹窗

```text
[弹窗标题：关闭/取消跟进商机]
  - 操作类型：关闭 / 取消跟进
  - 关闭类型，待字典配置
  - 原因，必填
  - 说明，必填
  - 是否可重启
  - 确认按钮 / 取消按钮
```

验收关注：

- 商机阶段和状态必须独立维护。
- 关闭或取消跟进必须填写原因。
- 关闭或取消跟进后，不进入默认跟进列表，不计入默认销售预测。
- 销售行动完成后，商机最近跟进和周进展汇总自动更新。

### 7.1 行动列表布局

```text
[顶部标题与操作]
  - 标题：销售行动
  - 主操作：新建行动

[筛选栏]
  - 行动状态

  - 关联客户
  - 关联商机
  - 责任人

[表格]
  - 行动主题

  - 关联商机
  - 责任人
  - 行动时间
  - 状态
  - 结果
  - 是否进入周进展
```

### 7.2 行动新建/编辑布局

```text
[基础信息]
  - 行动主题，必填
  - 行动类型，必填
  - 关联客户，必填
  - 关联商机，可选
  - 关联联系人，多选
  - 责任人，必填
  - 参与人员
  - 行动时间，必填
  - 行动状态

[过程记录]
  - 沟通内容
  - 客户反馈
  - 形成结论
  - 行动结果

[下一步]
  - 下一步计划
  - 下次跟进时间
  - 是否生成提醒

[风险与周进展]
  - 风险类型
  - 风险说明
  - 是否进入周进展
  - 周进展周期，默认按行动时间所在自然周

[附件/备注]
  - 附件入口
  - 备注
```

验收关注：

- 可以创建不关联商机的客户经营行动。
- 可以创建关联商机的项目推进行动。
- 行动完成后自动回写客户和商机最近跟进。
- 已完成且进入周进展的行动自动进入商机周进展。
- 行动结果为发现风险时，可更新商机风险状态。

## 8. 周进展

### 8.1 商机周进展汇总布局

```text
[筛选栏]
  - 自然周
  - 商机
  - 客户
  - 负责人
  - 风险状态

[商机分组]
  [商机摘要]
    - 商机名称
    - 客户
    - 阶段/状态
    - 负责人
    - 最近跟进

  [进展明细]
    - 行动时间
    - 行动主题
    - 形成结论
    - 下一步计划
    - 风险说明
    - 关联行动详情入口
```

### 8.2 负责人周进展汇总布局

```text
[筛选栏]
  - 部门
  - 负责人
  - 自然周
  - 客户
  - 商机

[负责人分组]
  [负责人摘要]
    - 本周行动数
    - 已完成行动数
    - 涉及商机数
    - 风险行动数

  [商机进展]
    - 按商机展示该负责人的行动明细
    - 支持下钻到商机详情和行动详情
```

验收关注：

- 周进展必须来自已完成销售行动。
- 同一商机一周多条行动保留多条明细，不强行合并。
- 支持按商机、负责人、自然周查看。
- 商机主表不存储第一周、第二周、第三周、第四周等周进展字段。

## 9. 系统管理

### 9.1 用户管理布局

```text
[筛选栏]
  - 姓名/账号
  - 部门
  - 角色
  - 状态

[表格]
  - 姓名
  - 账号
  - 部门

  - 手机/邮箱
  - 状态
  - 最近登录
```

验收关注：

- 管理员可创建、编辑、启用、停用用户。
- 停用用户无法登录。
- 管理员可重置密码。

### 9.2 组织管理布局

```text
[左侧组织树]
  - 公司
  - 部门
  - 团队

[右侧详情]
  - 组织名称
  - 上级组织
  - 组织类型
  - 负责人
  - 区域
  - 状态
  - 关联用户列表
```

验收关注：

- 组织层级可维护。
- 用户可绑定组织。
- 数据权限可按组织范围引用。

### 9.3 角色权限布局

```text
[左侧角色列表]
  - 销售个人
  - 销售负责人
  - 管理层
  - 管理员
  - 其他角色待确认

[右侧权限配置]
  [菜单权限]
    - 工作台、客户池、联系人、商机、销售行动、周进展、系统管理

  [操作权限]
    - 新增、编辑、删除、导出、关闭商机、配置权限等

  [数据范围]
    - 本人
    - 本部门
    - 本部门及下级
    - 协同数据
    - 全局
```

验收关注：

- 不同角色登录后看到的菜单和按钮不同。
- 销售个人只能访问本人或协同数据。
- 销售负责人可访问本部门数据。
- 管理层可访问全局数据。
- 权限变更写入审计日志。

## 10. 关键链路低保真串联

```text
用户登录
  -> 工作台
  -> 新建客户
  -> 客户详情中新建联系人
  -> 客户详情中新建商机
  -> 商机详情中新建销售行动
  -> 完成销售行动
  -> 客户和商机最近跟进自动回写
  -> 商机周进展汇总展示行动形成结论、下一步计划和风险说明
```
