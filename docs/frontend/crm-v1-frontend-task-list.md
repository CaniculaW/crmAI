# CRM V1 前端开发任务清单

日期：2026-06-17

适用范围：按研发启动计划Task 7拆分前端Sprint任务。默认技术栈参考 `docs/architecture/tech-stack-decision.md`：React + Ant Design。

## 1. 前端执行原则

- V1优先交付可用的企业后台工作流，不做营销型页面。
- 页面字段以PRD和API清单为准，枚举从字典接口读取。
- 菜单和按钮按权限动态展示，但所有权限仍以后端校验为准。
- 列表、筛选、分页、详情Tab、表单弹窗保持一致交互。

## 4. 状态管理建议

| 状态类型 | 建议处理 |
|---|---|
| 登录态 | 保存当前用户、角色、权限点、数据范围；刷新时调用 `/api/auth/me` 恢复。 |
| 服务端列表数据 | 使用请求状态管理，按查询参数缓存；新增/编辑/完成后失效对应列表和详情缓存。 |
| 字典数据 | 登录后或页面首次进入时加载，可缓存；字典变更后刷新。 |
| 表单临时状态 | 使用Ant Design Form局部管理，提交成功后关闭弹窗或抽屉。 |
| 权限状态 | 菜单和按钮根据权限点展示；接口403时统一提示并刷新权限。 |

## 6. 页面验收口径

### 6.1 工程骨架

- 本地可启动。
- 支持路由。
- 支持登录态保存和退出。
- 支持统一请求拦截和错误提示。

### 6.2 登录与系统管理

- 登录页可登录。
- 修改密码可用。
- 用户、组织、角色、字典配置页面可用。
- 权限不同，菜单展示不同。

### 6.3 客户池

- 客户列表支持筛选和分页。
- 客户表单字段完整。
- 客户详情展示联系人、商机、行动入口。

### 6.4 联系人

- 联系人列表支持客户、类型、态度、关系热度筛选。
- 联系人表单字段完整。
- 客户关系地图或关系分组视图可用。

### 6.5 商机

- 商机列表支持阶段、状态、负责人、金额、风险筛选。
- 商机详情展示基础信息、干系人、行动、周进展、风险。
- 商机关闭/取消跟进弹窗强制填写原因。

### 6.6 销售行动

- 行动列表支持状态、类型、客户、商机、责任人筛选。
- 行动表单支持关联客户、商机、联系人。
- 行动完成后前端刷新客户和商机最近跟进。

### 6.7 周进展

- 可按商机、负责人、自然周查看周进展。
- 周进展展示行动时间、行动主题、形成结论、下一步计划、风险说明。
