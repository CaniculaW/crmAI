# 项目型大客户CRM V1测试用例

版本：V1.0

适用范围：Sprint 0测试设计，覆盖V1销售基础闭环的系统基础、客户、联系人、商机、销售行动、周进展与审计能力。

参考文档：

- `docs/superpowers/plans/2026-06-17-crm-development-kickoff.md`
- `docs/prd/project-crm-prd.md`
- `docs/superpowers/specs/2026-06-17-project-crm-design.md`
- `docs/database/project-crm-data-model.md`

## 1. 测试范围

本用例覆盖启动计划 Task 8 中要求的范围：

- 登录、登出、修改密码、重置密码。
- 用户、组织、角色、权限、数据范围。
- 客户新增、编辑、查询、详情。
- 联系人新增、编辑、关系视图。
- 商机新增、阶段变更、状态变更、关闭、取消跟进。
- 销售行动新增、完成、风险触发、最近跟进回写。
- 商机周进展自动汇总。
- 核心集成链路：登录 -> 新建客户 -> 新建联系人 -> 新建商机 -> 新建销售行动 -> 完成销售行动 -> 查看商机周进展 -> 关闭商机。

## 2. 测试数据与角色准备

| 数据类型 | 建议准备 |
|---|---|
| 组织 | 至少2个同级销售部门、1个上级部门、1个下级部门 |
| 用户 | 管理员、销售个人A、销售个人B、销售负责人、管理层、售前/方案人员 |
| 角色 | 管理员、销售个人、销售负责人、管理层、售前/方案人员 |
| 数据范围 | 本人、本部门、本部门及下级、协同数据、全局 |
| 客户 | 本人负责客户、同部门其他销售负责客户、跨部门客户、协同客户、上级/下级客户 |
| 联系人 | 不同客户下的决策人、技术负责人、采购负责人、财务负责人、外部伙伴 |
| 商机 | 跟进中商机、暂停商机、关闭商机、取消跟进商机、高风险商机 |
| 销售行动 | 待完成行动、已完成行动、发现风险行动、不进入周进展行动、未关联商机的客户经营行动 |

| 用例编号 | 测试场景 | 前置条件 | 测试步骤 | 预期结果 | 优先级 | 测试类型 |
|---|---|---|---|---|---|---|
| AUTH-001 | 正常登录 | 用户状态启用，账号状态启用，密码正确 | 1. 打开登录页；2. 输入账号和密码；3. 点击登录 | 登录成功，进入系统首页；系统加载该用户菜单、角色权限和数据范围；产生登录成功日志 | P0 | 功能 |
| AUTH-002 | 密码错误登录失败 | 用户状态启用 | 1. 输入正确账号和错误密码；2. 点击登录 | 登录失败并提示账号或密码错误；不建立会话；记录登录失败日志 | P0 | 异常 |
| AUTH-003 | 停用用户不得登录 | 用户状态为停用 | 1. 使用停用用户账号登录 | 登录失败并提示账号不可用或已停用；不建立会话 | P0 | 权限 |
| AUTH-004 | 登出后会话失效 | 用户已登录 | 1. 点击登出；2. 使用原会话访问受保护页面或接口 | 用户退出到登录页；原会话失效；受保护资源要求重新登录；记录登出日志 | P0 | 功能 |
| AUTH-005 | 会话过期后重新登录 | 会话过期时间已配置 | 1. 登录系统；2. 等待或模拟会话过期；3. 访问业务页面 | 系统提示登录态失效；用户需重新登录；不得继续访问业务数据 | P0 | 回归 |
| AUTH-006 | 用户修改密码成功 | 用户已登录，知道旧密码 | 1. 进入修改密码页；2. 输入旧密码、新密码和确认密码；3. 提交 | 修改成功；旧密码失效；新密码可登录；记录修改密码审计日志 | P0 | 功能 |
| AUTH-007 | 修改密码旧密码错误 | 用户已登录 | 1. 进入修改密码页；2. 输入错误旧密码；3. 提交 | 修改失败；提示旧密码不正确；密码不变；记录失败或异常事件 | P0 | 异常 |
| AUTH-008 | 修改密码不符合强度 | 用户已登录，密码强度规则已配置 | 1. 输入弱密码或确认密码不一致；2. 提交 | 前端或后端拦截；提示密码强度或确认密码错误；密码不变 | P0 | 异常 |
| AUTH-009 | 管理员重置密码 | 管理员已登录，目标用户启用 | 1. 管理员进入用户管理；2. 选择目标用户；3. 执行重置密码 | 重置成功；目标用户使用重置密码可登录；首次登录提示修改密码；记录管理员、目标用户和操作内容 | P0 | 权限 |
| AUTH-010 | 非管理员不得重置密码 | 销售个人已登录 | 1. 访问重置密码入口或调用重置接口 | 无入口或接口返回权限不足；目标用户密码不变；记录越权访问事件 | P0 | 权限 |

| IAM-006 | 配置角色操作权限 | 管理员已登录，角色存在 | 1. 取消销售个人对某模块的新建权限；2. 用销售个人访问该模块 | 新建按钮不可见或不可用；直接调用新建接口返回权限不足 | P0 | 权限 |

| IAM-008 | 本人数据范围 | 销售个人A、销售个人B分别拥有客户和商机 | 1. 销售个人A登录；2. 查看客户、联系人、商机、销售行动列表 | 只能看到本人负责或协同/参与的数据；不能看到无关销售个人B数据 | P0 | 权限 |
| IAM-009 | 本部门数据范围 | 销售负责人已绑定销售部门 | 1. 销售负责人登录；2. 查看本部门客户、商机、行动；3. 查看跨部门无关数据 | 可查看本部门数据；不可查看无关跨部门数据 | P0 | 权限 |
| IAM-010 | 全局数据范围 | 管理层或管理员具备全局范围 | 1. 管理层登录；2. 查看不同部门客户、商机、行动 | 可查看全局业务数据；操作按钮仍受角色操作权限约束 | P1 | 权限 |
