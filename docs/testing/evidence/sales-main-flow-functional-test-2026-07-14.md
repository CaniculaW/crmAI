## 4. 缺陷台账

### FT-SALES-DEF-001 P2 客户不存在时返回 500

- 页面/API：`GET /api/accounts/999999999`
- 步骤：管理员登录后请求不存在的客户 ID。
- 预期：HTTP 404，返回可理解的“客户不存在”业务错误。
- 实际：HTTP 500，`code=INTERNAL_ERROR`。
- 证据：`trace_id=FT-SALES-0714-ACCOUNT-NOTFOUND`。
- 复现率：1/1。
- 影响：页面访问失效或过期客户链接时只能显示服务端异常，无法区分对象不存在。

### FT-SALES-DEF-002 P2 已完成行动仍可再次完成

- 页面/API：`POST /api/activities/2/complete`
- 步骤：创建行动并完成；对同一行动再次调用完成接口。
- 预期：HTTP 409，拒绝非法重复状态迁移，或明确声明并验证幂等语义。
- 实际：HTTP 200，`code=OK`；服务会再次更新完成时间、版本、审计和业务回填。
- 证据：`trace_id=FT-SALES-0714-ACT-RECOMPLETE`，对象 ID `2`。
- 复现率：1/1。
- 影响：可能生成重复审计和重复业务副作用，状态机与页面禁用规则不一致。

### FT-SALES-DEF-003 P2 非法周进展月份返回 500

- 页面/API：`GET /api/weekly-progress/opportunities?month=bad-month`
- 步骤：传入不符合 `yyyy-MM` 的月份参数。
- 预期：HTTP 400，提示月份格式错误。
- 实际：HTTP 500，`code=INTERNAL_ERROR`。
- 证据：`trace_id=FT-SALES-0714-WEEKLY-INVALID-MONTH`。
- 复现率：1/1。
- 影响：错误筛选参数被包装成服务端故障，前端无法给出可操作反馈。

### FT-SALES-DEF-004 P2 仪表盘接受倒置日期范围

- 页面/API：`GET /api/dashboard/funnel?date_from=2026-07-20&date_to=2026-07-01`
- 步骤：开始日期晚于结束日期。
- 预期：HTTP 400 或 422，提示日期范围无效。
- 实际：HTTP 200，返回空统计；响应仍保留倒置的过滤条件。
- 证据：`trace_id=FT-SALES-0714-DASH-INVERTED-DETAIL`。
- 复现率：1/1。
- 影响：用户会把参数错误误判为真实的零业务数据。
