# CRM V1.0.0-rc.7 候选版本记录

计划 Git tag：

```text
v1.0.0-rc.7
```

## 1. 候选版本定位

`v1.0.0-rc.7` 是在 `v1.0.0-rc.6` 基础上补强 UAT 证据包 Go/No-Go 准出一致性校验的 V1 候选版本，用于继续推进正式 UAT 和试点准出。

该版本包含：

- `v1.0.0-rc.6` 已具备的 RC/UAT readiness 审计、本地具名验证环境证据、UAT 证据包生成器、镜像源覆盖配置检查和远端 `V1 Validation` 质量门。
- `scripts/v1-uat-evidence-pack-validate.mjs`：读取已填写的 UAT 证据包，校验自动化结果、环境 Smoke、UAT-001 至 UAT-010、P0/P1 缺陷、Go/No-Go 判定和签署记录是否一致。
- `scripts/v1-uat-evidence-pack-validate.test.mjs`：覆盖完整 `Go` 证据包、P0 未关闭时误选 `Go`、业务签署缺失时误选 `Go` 的失败场景。
- `scripts/v1-uat-evidence-pack.mjs` 和 UAT 模板新增 validator 输出记录区。
- GitHub Actions `V1 Validation` 增加 UAT 证据包 validator 测试。

该版本已完成研发侧自动化验证、部署配置校验、RC/UAT 就绪审计和本地服务级具名环境验证；仍需在项目指定具名测试环境完成 UI 级浏览器 Smoke、UAT 演示、缺陷闭环和销售侧/管理侧验收签署后，才能升级为正式 V1 发布版本。

## 2. V1范围

本候选版本覆盖：

- 登录、登出、修改密码、管理员重置密码。
- 组织、用户、角色、权限、数据范围和字典维护。
- 客户池、联系人与干系人、商机生命周期、销售行动、提醒待办、附件元数据。
- 销售行动完成后回写客户和商机最近跟进。
- 商机周进展按已完成销售行动自动汇总。
- 关键操作审计日志。
- V1演示管理员种子和部署态 bootstrap 探针。
- 前端 V1 基础页面、核心对象筛选、详情入口和主要编辑操作。
- UAT 证据包草稿生成器和 Go/No-Go validator。
- 测试环境镜像源覆盖配置。

本候选版本不包含：

- 方案标书。
- 合同管理。
- 开票和回款管理。
- 经营驾驶舱完整指标。
- AI销售助手完整能力。

## 6. UAT证据包生成与校验

生成草稿：

```bash
node scripts/v1-uat-evidence-pack.mjs \
  --environment <test-env-name> \
  --frontend-url <test-frontend-url> \
  --backend-url <test-backend-url> \
  --git-commit <git-sha> \
  --rc v1.0.0-rc.7 \
  --username <masked-admin-user> \
  > crm-v1-uat-evidence-pack.md
```

填写完成后校验：

```bash
node scripts/v1-uat-evidence-pack-validate.mjs crm-v1-uat-evidence-pack.md
```

若选择 `Go`，validator 必须返回 `PASS`。生成器和 validator 都不替代真实测试执行、截图归档、缺陷闭环和业务签署。

## 7. 已知限制

- 当前机器访问 Docker Hub token 接口曾超时，因此本地未完成 `docker compose -f compose.v1-test.yml up -d --build` 的完整后端/前端镜像启动；本版本已提供可配置镜像源入口，具名测试环境可切换企业镜像代理后执行完整构建。
