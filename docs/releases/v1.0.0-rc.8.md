## 1. 候选版本定位

`v1.0.0-rc.8` 是在 `v1.0.0-rc.7` 基础上补强 V1 测试环境演示业务数据的候选版本，用于继续推进正式 UAT 和试点准出。

该版本包含：

- `v1.0.0-rc.7` 已具备的 RC/UAT readiness 审计、本地具名验证环境证据、UAT 证据包生成器、UAT 证据包 Go/No-Go validator、镜像源覆盖配置检查和远端 `V1 Validation` 质量门。
- `V1DemoDataSeeder` 在启用 `CRM_SEED_V1_DEMO_ENABLED=true` 时，除演示管理员、角色、权限和全局数据范围外，自动准备 V1演示业务数据：客户、联系人、商机、已完成销售行动、对象关联关系和周进展聚合来源数据。
- `V1DemoDataSeederTest` 校验 `/api/bootstrap` 中 `accounts`、`contacts`、`opportunities`、`activities` 非空，校验 `v_opportunity_weekly_progress` 可聚合出记录，并覆盖重复执行 seed 不产生重复演示业务数据。
- `scripts/v1-uat-readiness-check.mjs` 将 rc.8 候选版本、本地 UAT 证据版本一致性、V1演示业务数据非空计数和 rc.8 UAT 交接草稿 No-Go 阻塞项纳入 readiness 审计，业务对象计数按 `>= 1` 校验。
- `scripts/v1-release-gate.mjs` 将 RC/UAT readiness、UAT 具名环境证据 validator、正式 UAT 证据包 validator、UAT 证据清单 validator、UAT 执行追踪表 validator、UAT 缺陷台账 validator、UAT 签署台账 validator 和项目 `Go` 决策合并为最终放行门禁；当前 rc.8 草稿返回 `FAIL / No-Go`，符合未完成外部验收的预期。
- `docs/testing/crm-v1-uat-execution-tracker.md` 将 PRE-001 至 PRE-006、SMK-001 至 SMK-005、UAT-001 至 UAT-010、缺陷闭环、签署角色和最终 release gate 组织成可派工追踪表。
- `docs/testing/v1-uat-environment-evidence.md` 将具名测试环境名称、前后端地址、提交号、ENV-001 至 ENV-008 Smoke/账号/权限证据集中为可校验材料；当前保持 `No-Go`，直到项目指定测试环境证据补齐。
- `docs/testing/v1-uat-evidence-manifest.md` 将 PRE、SMK、UAT、P0/P1缺陷、签署和 Go/No-Go 证据引用集中为可校验清单；当前清单保持 `No-Go`，直到外部证据补齐。
- `docs/testing/v1-uat-defect-register.md` 将 P0/P1 缺陷闭环、缺陷明细、回归证据和项目准出结论集中为可校验台账；当前台账保持 `No-Go`，直到测试侧补齐缺陷和回归证据。
- `docs/testing/v1-uat-signoff-register.md` 将销售侧、管理侧、产品、测试、研发和项目负责人签署结论集中为可校验台账；当前台账保持 `No-Go`，直到六方具名签署、签署日期、证据引用和项目 `Go` 结论补齐。
- `scripts/v1-uat-execution-tracker-validate.mjs` 校验 UAT 执行追踪表的角色、PRE、SMK、UAT、P0/P1、签署和 release gate 状态；当前 rc.8 追踪表返回 `FAIL / No-Go`，用于持续暴露外部验收阻塞项。
- `scripts/v1-uat-environment-validate.mjs` 校验具名测试环境证据必须覆盖环境元数据、ENV-001 至 ENV-008、证据引用、责任人和敏感材料；当前草稿返回 `FAIL / No-Go`。
- `scripts/v1-uat-evidence-manifest-validate.mjs` 校验证据清单必须覆盖全部证据项、PASS项必须有具体引用且不得含明文敏感材料；当前草稿返回 `FAIL / No-Go`。
- `scripts/v1-uat-defect-register-validate.mjs` 校验缺陷台账必须覆盖 P0/P1 汇总、关闭状态、回归证据、敏感材料和 Go/No-Go 结论；当前草稿返回 `FAIL / No-Go`。
- `scripts/v1-uat-signoff-register-validate.mjs` 校验签署台账必须覆盖六方签署、签署日期、证据引用、敏感材料和项目 `Go` 结论；当前草稿返回 `FAIL / No-Go`。
- `scripts/v1-validation-status.mjs` 聚合 readiness、UAT证据包、UAT执行追踪表、UAT签署台账和 release gate 输出 `docs/testing/v1-validation-status.md`，作为项目例会和验收推进的当前状态页。
- `scripts/v1-uat-action-plan.mjs` 将 validators 的当前阻塞项转换为项目/产品、测试、业务UAT和研发四条执行工作流，输出 `docs/testing/v1-uat-action-plan.md`。
- `scripts/v1-uat-execution-pack.mjs` 将失败门禁拆成 ENV、PRE、SMK、UAT、DEF、SIGNOFF 和 GO-NOGO 逐项补证清单，输出 `docs/testing/v1-uat-execution-pack.md`；该清单用于外部 UAT 执行推进，不替代正式 PASS 证据。
- `scripts/v1-go-no-go-meeting.mjs` 生成 V1 Go/No-Go 会议包，集中列出参会角色、门禁命令、开放阻塞项和最终签署表，输出 `docs/testing/v1-go-no-go-meeting.md`。

该版本已完成研发侧自动化验证、部署配置校验、RC/UAT 就绪审计、本地服务级具名环境验证和 Compose 部署态工程验证；仍需在项目指定具名测试环境完成 UAT 演示、缺陷闭环和销售侧/管理侧验收签署后，才能升级为正式 V1 发布版本。

## 2. V1范围

本候选版本覆盖：

- 登录、登出、修改密码、管理员重置密码。
- 组织、用户、角色、权限、数据范围和字典维护。
- 客户池、联系人与干系人、商机生命周期、销售行动、提醒待办、附件元数据。
- 销售行动完成后回写客户和商机最近跟进。
- 商机周进展按已完成销售行动自动汇总。
- 关键操作审计日志。
- V1演示管理员种子、V1演示业务数据和部署态 bootstrap 探针。
- 前端 V1 基础页面、核心对象筛选、详情入口和主要编辑操作。
- UAT 证据包草稿生成器和 Go/No-Go validator。
- 测试环境镜像源覆盖配置。

本候选版本不包含：

- 方案标书。
- 合同管理。
- 开票和回款管理。
- 经营驾驶舱完整指标。
- AI销售助手完整能力。

## 6. UAT证据包生成与校验

生成草稿：

```bash
node scripts/v1-uat-evidence-pack.mjs \
  --environment <test-env-name> \
  --frontend-url <test-frontend-url> \
  --backend-url <test-backend-url> \
  --git-commit <git-sha> \
  --rc v1.0.0-rc.8 \
  --username <masked-admin-user> \
  > crm-v1-uat-evidence-pack.md
```

```bash
node scripts/v1-uat-evidence-pack-validate.mjs crm-v1-uat-evidence-pack.md
```
